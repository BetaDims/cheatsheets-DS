Closures & ámbito (scope)

A

Una closure es una función que "recuerda" las variables del entorno donde se creó, aunque ese entorno ya haya terminado. Es la base de la privacidad y de muchos patrones en JS.

Scopeconst/let tienen ámbito de bloque (entre { }). Declara las variables lo más cerca posible de donde se usan.

El event loop — por qué JS no se congela

A

JS ejecuta en un solo hilo, pero no se bloquea esperando (una respuesta de red, un temporizador). Delega esas tareas y, cuando terminan, su callback entra en cola; el event loop la ejecuta cuando el hilo queda libre.

1 · Call stack

el código que corre ahora, línea a línea.

2 · Web APIs

tareas que el navegador maneja aparte: fetch, setTimeout.

3 · Cola de tareas

los callbacks ya listos esperan su turno.

4 · Event loop

cuando el stack está vacío, pasa el siguiente callback.

Por eso importaNunca hagas trabajo pesado y síncrono en el hilo principal: congela la interfaz. Usa código asíncrono para esperas.

Promesas & async/await

A

Una promesa representa un valor futuro. async/await permite escribir código asíncrono que se lee como síncrono.

fetch — pedir datos a una API

I

fetch devuelve una promesa con la respuesta HTTP. Lo normal: await + .json().
