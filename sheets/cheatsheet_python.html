Ejecutar Python & el REPL

B

Dos formas de correr código: un script (python archivo.py) o el REPL interactivo (python sin más), donde escribes una línea y ves el resultado al instante — ideal para probar y explorar. >>> es el prompt; lo que devuelve aparece debajo.

Python 3.13 · consola interactiva
>>> 2 ** 10
1024
>>> nombre = "Ada"
>>> len(nombre)
3
>>> print(f"Hola, {nombre}!")
Hola, Ada!
>>> type(3.14)
<class 'float'>
>>>
Atajos claveEn el REPL, _ guarda el último resultado · help(obj) abre la ayuda · dir(obj) lista atributos y métodos · Ctrl-D (o exit()) sale.

Variables & tipos básicos

B

No se declara el tipo: la variable es una etiqueta que apunta a un objeto. = asigna; == compara. Nombres en snake_case.

Tipado dinámico, fuerteEl tipo lo lleva el objeto, no la variable, pero Python no mezcla tipos a la fuerza: "3" + 5 es un error, no "35" ni 8.

Por qué importa% revela paridad (n % 2 == 0 = par) y // reparte sin decimales. int no tiene límite de tamaño; float sí redondea (0.1 + 0.2 ≠ 0.3 exacto).

Conversión de tipos & entrada del usuario

B

Convierte explícitamente con las funciones del tipo. input() siempre devuelve str: si esperas un número, conviértelo o fallarán los cálculos.

Error típicoinput() + número sin convertir: "5" + 1 lanza TypeError. Envuelve siempre en int(...) o float(...).

Índices & rebanadas (slicing)

B

Cada carácter tiene un índice desde 0; los negativos cuentan desde el final. La rebanada [inicio:fin:paso] incluye inicio y excluye fin.

RecuerdaEl slicing funciona igual en listas y tuplas. Las cadenas son inmutables: s[0]="x" falla; crea una nueva con métodos.

f-strings & formato de impresión

B

La f-string (prefijo f) inserta variables y expresiones entre {}. Es la forma moderna y legible de construir texto; además da control de formato (decimales, alineación, miles).

Mini-lenguaje de formato{valor:[relleno][align][,][.precisión][tipo]}. Tipos útiles: f fijo, % porcentaje, e científico, d entero, x hex.
02

Estructuras de datos

list · tuple
dict · set

Listas — ordenadas y mutables

B

Secuencia ordenada que puede crecer, encoger y cambiar. El caballo de batalla de Python. Se crea con [].

Tuplas — ordenadas e inmutables

B

Como una lista pero no cambia tras crearse. Se crea con (). Útil para datos fijos (coordenadas, un registro) y para devolver varios valores de una función.

Diccionarios — pares clave→valor

I

Asocia una clave única a un valor; acceso instantáneo por clave. La estructura más usada para datos estructurados (como un JSON). Se crea con {}.

Conjuntos — únicos, sin orden

I

Colección de elementos únicos, sin duplicados ni orden. Perfecto para quitar repetidos y para operaciones de conjuntos. Se crea con {} (con elementos) o set().

¿Cuál estructura uso?

I

RendimientoBuscar un elemento (x in …) es instantáneo en set y dict (tabla hash) pero lento en list (recorre todo). *Los dict conservan el orden de inserción desde Python 3.7.

if / elif / else

B

El bloque se define por la indentación (4 espacios), no por llaves. El : abre el bloque.

Bucles for & while

B

for recorre cualquier iterable (lista, texto, dict…). while repite mientras una condición sea verdadera. break sale; continue salta a la siguiente vuelta.

Comprehensions — construir colecciones en una línea

I

Forma idiomática y rápida de crear listas, dicts o sets a partir de un iterable, con filtro opcional. Lee como inglés: "x al cuadrado, para cada x en el rango, si x es par".

No te pasesSi una comprehension necesita varios if/for anidados y deja de leerse de un vistazo, usa un for normal. Legible > ingenioso.
04

Funciones

def · return

def & parámetros

B

Empaqueta lógica reutilizable. Los parámetros pueden tener valor por defecto y pasarse por nombre (keyword), lo que hace la llamada legible.

Trampa clásicaNunca uses una lista/dict como valor por defecto (def f(x=[])): se comparte entre llamadas. Usa x=None y créala dentro.

*args & **kwargs

I

*args recoge posicionales extra en una tupla; **kwargs recoge keywords extra en un dict. Para funciones que aceptan un número variable de argumentos.

lambda · map · filter

I

Una lambda es una función anónima de una sola expresión. Brilla como argumento key= o con map/filter (aunque a menudo una comprehension es más clara).

Alcance de variables (LEGB)

A

Python busca un nombre en este orden: Local → Enclosing (función externa) → Global → Built-in. Una función puede leer globales, pero para reasignarlas necesita global / nonlocal.

Localdefinida dentro de la función; muere al terminar.
Enclosingde una función que envuelve a otra (closures).
Globala nivel de módulo; usa global x para reasignar.
Built-inlen, print, range… siempre disponibles.
Buena prácticaEvita global: pasa datos por parámetros y devuelve resultados con return. Funciones puras = más fáciles de probar.

Anotaciones de tipo (type hints)

I

Opcionales y no obligan en tiempo de ejecución, pero documentan la intención y permiten que el editor y herramientas como mypy detecten errores antes de correr. Hoy son estándar en código profesional.

Clases & objetos

I

Una clase es el molde; un objeto (instancia) es lo que creas con él. __init__ es el constructor: corre al crear el objeto. self es la instancia actual — el primer parámetro de todo método.

Herencia & polimorfismo

A

Una subclase hereda y puede sobrescribir métodos del padre. super() llama al método del padre. Polimorfismo: distintas clases responden al mismo método a su manera.

Métodos dunder (mágicos)

A

Los métodos __xxx__ conectan tu objeto con la sintaxis de Python: cómo se imprime, se suma, se compara o se mide su longitud.

MétodoSe activa con
__init__crear: Clase()
__str__print(obj), str()
__repr__el REPL, depuración
__len__len(obj)
__eq__obj == otro
__add__obj + otro

@property & @dataclass

A

@property expone un método como si fuera un atributo (cálculo al vuelo, validación). @dataclass genera __init__, __repr__ y __eq__ por ti para clases que solo guardan datos.

import & módulos

B

Un módulo es un archivo .py; un paquete es una carpeta de módulos. import trae código de la biblioteca estándar, de paquetes instalados o tuyos.

Evitafrom modulo import * ensucia el espacio de nombres y oculta de dónde viene cada cosa. Importa lo concreto.

if __name__ == "__main__"

I

El código bajo este if solo corre cuando ejecutas el archivo directamente, no cuando otro lo importa. Separa "soy un programa" de "soy una biblioteca".

pip & entornos virtuales

I

Un entorno virtual aísla las dependencias de cada proyecto para que no choquen entre sí. Créalo, actívalo, instala con pip y congela versiones en requirements.txt. Comandos para tu terminal (zsh):

Regla de oroUn entorno por proyecto y .venv/ en el .gitignore. Lo que se versiona es requirements.txt, no la carpeta del entorno.

Errores & excepciones

try / except / else / finally

I

Captura errores previsibles en vez de dejar que el programa muera. else corre si no hubo error; finally corre siempre (cerrar archivos, liberar recursos).

raise & excepciones propias

I

raise lanza un error a propósito (validar entradas). Define los tuyos heredando de Exception para errores con significado en tu dominio.

08

Avanzado & biblioteca estándar

with · yield
decoradores · stdlib

Context managers (with)

I

with garantiza que un recurso se cierre aunque haya un error: abre, te deja trabajar y limpia al salir del bloque. La forma correcta de leer/escribir archivos.

Generadores (yield)

A

Una función con yield devuelve valores de a uno y bajo demanda, sin construir toda la lista en memoria. Ideal para secuencias grandes o infinitas.

Decoradores

A

Un decorador envuelve una función para añadirle comportamiento (logging, medir tiempo, cachear, controlar acceso) sin tocar su código. La sintaxis @deco encima de la función es azúcar para fn = deco(fn).

En la prácticaDecoradores que usarás listos: @property, @staticmethod, @classmethod, @functools.cache (memoiza resultados) y los de frameworks (@app.route en Flask).
