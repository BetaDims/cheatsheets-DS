El documento base

B

Toda página parte de este esqueleto: declaración del tipo, idioma, metadatos en <head> y contenido visible en <body>.

Qué va en el <head>

B

No se ve, pero importa: metadatos, enlaces a CSS, fuentes e info para buscadores y redes.

EtiquetaPara qué

<link rel="stylesheet">conectar el CSS

Anatomía de una etiqueta

B

Una etiqueta (elemento) tiene apertura, atributos opcionales, contenido y cierre. Los atributos configuran su comportamiento; el contenido es lo que envuelve.

<a href="/inicio">apertura + atributo Inicio contenido</a>cierre
Elementoapertura + contenido + cierre. Algunos son vacíos (sin cierre): <img>, <br>, <input>.
Atributopar nombre="valor" en la apertura: id, class, href, src, alt…
class vs idclass se repite (estilo de muchos); id es único por página.

Texto & listas

B

Usa la etiqueta por su significado, no por cómo se ve (de eso se encarga CSS). Hay un solo <h1> por página; los títulos bajan en orden.

Enlaces & imágenes

B

<a> enlaza con href; <img> incrusta con src y siempre con alt (accesibilidad y SEO).

Tablas

I

Para datos tabulares (no para maquetar). <thead> y <tbody> estructuran; <th> son cabeceras.

Etiquetas semánticas

I

En vez de <div> para todo, usa etiquetas que describen su rol. Ayudan a SEO, accesibilidad y a que el código se lea solo.

Accesibilidad (a11y)

I

Hacer la web usable para todas las personas (incluidos lectores de pantalla) suele ser, además, mejor HTML.

alt en imgdescribe la imagen; vacío (alt="") si es decorativa.
labeltodo campo de formulario con su etiqueta.
jerarquíatítulos en orden (h1→h2→h3), sin saltos.
contrastetexto legible sobre el fondo (WCAG AA).
aria-*atributos para casos que el HTML no cubre (aria-label, roles).
Regla prácticaUsa el elemento nativo correcto (<button>, <a>) antes que recrearlo con <div> + ARIA. Lo nativo ya es accesible.

La cascada & la especificidad

I

Cuando varias reglas chocan, gana la de mayor especificidad; a igualdad, la última en el archivo. Por eso un #id vence a una .clase, y esta a una etiqueta.

En líneastyle="…" — la más fuerte (1000). Evítala.
#idpeso alto (100). Úsalo con moderación.
.clase / :pseudo / [attr]peso medio (10). El nivel ideal para estilar.
etiquetapeso bajo (1).
!important!important rompe la cascada y es difícil de revertir. Es señal de que la especificidad se te fue de las manos: casi siempre hay una forma mejor.

Box model

B

Cada elemento es una caja con cuatro capas, de dentro a fuera: contenido → padding → border → margin.

border
padding
contenido
box-sizing* { box-sizing: border-box; } hace que width incluya padding y border. Es el primer reset que pone todo el mundo — sin él, las medidas se descuadran.

display & espaciado

B
PropiedadHace

display:inlinefluye en el texto (span, a)
display:inline-blocken línea pero con tamaño

margin vs paddingMargin = espacio fuera (separa de vecinos). Padding = espacio dentro (entre borde y contenido).

Flexbox en vivo — alinear en una fila o columna

I

Flexbox reparte espacio entre elementos en un eje (fila o columna). El padre es display:flex; los hijos se alinean con justify-content (eje principal) y align-items (eje cruzado). Cambia el reparto y míralo en vivo:

render · display:flex · justify-content: flex-start
1
2
3
Cuándo FlexboxPara alinear cosas en una línea: barras de navegación, una fila de tarjetas, centrar un elemento (justify-content:center; align-items:center).

CSS Grid en vivo — filas y columnas a la vez

I

Grid maqueta en dos ejes (filas y columnas) a la vez. Defines las pistas con grid-template-columns y la unidad fr (fracción del espacio libre). Esto es un grid real renderizado:

render · grid-template-columns: 1fr 1fr 1fr

Flex o GridFlex para una dimensión (una fila/columna); Grid para layouts de dos dimensiones (galerías, la estructura de toda la página). Se combinan sin problema.

Media queries

I

Aplican estilos según el tamaño de pantalla. El enfoque mobile-first: estilo base para móvil y min-width para ir ampliando.

Posición & transición

A

Variables CSS (custom properties)

I

Define valores reutilizables una vez (en :root) y úsalos con var(). Cambiar el tema es cambiar una línea — la base de los design tokens.

Cierre del flujoHTML marca el significado, CSS la presentación. Mantenlos separados, usa clases (no !important), piensa mobile-first y deja que Flexbox y Grid hagan el layout.

CSS moderno

lo que llegó
en 2023–2026

:has() — el selector de «padre»

I

Por fin un selector que estiliza un elemento según lo que contiene. Marca la casilla y observa cómo cambia la tarjeta (el padre), sin una línea de JS:

cuándoFormularios reactivos sin JS, layouts que cambian si hay imagen (.card:has(img)) o validación visual al instante.

:is() & :where()

I

Agrupan selectores y evitan repetir. La diferencia: :is() suma especificidad; :where() vale siempre 0 (ideal para estilos base fáciles de sobrescribir).

Anidación nativa (nesting)

I

Anida reglas como en Sass, pero sin compilador. El símbolo & referencia al selector padre.

Container queries

A

Adapta un componente al ancho de su contenedor, no al de la pantalla. El mismo card se ve bien en una barra lateral o a pantalla completa.

Capas en cascada (@layer)

A

Ordena la prioridad de la cascada por capas, sin pelear con la especificidad. El orden en que las declaras manda: la última capa gana.

Subgrid

A

Un grid hijo hereda las pistas del padre, así varias tarjetas alinean sus títulos y pies aunque su texto tenga distinto largo.

para quéAlinear maquetas anidadas con la rejilla global: la pieza que faltaba para layouts realmente consistentes.

Propiedades lógicas

I

Hablan de inline (dirección del texto) y block en vez de left/top. Se adaptan solas a idiomas RTL o escritura vertical.

@property

A

Registra una variable CSS con tipo y valor inicial. Así el navegador puede animar custom properties (p. ej. el ángulo de un degradado).

View transitions

A

Anima el paso entre dos estados o páginas (como una app nativa) con muy poco código. Una línea de CSS activa las transiciones entre páginas del sitio.
