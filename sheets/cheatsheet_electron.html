01

Arquitectura: proceso main ⇄ renderer

B · concepto clave

Dos procesos, una app

B

Electron une Node.js y Chromium. Toda app tiene un único proceso main (arranca con main.js, controla ventanas y el sistema) y uno o más procesos renderer (cada ventana = una página web con su propio DOM).

El renderer no debe tocar el sistema directamente: pide cosas al main mediante IPC, a través de un script preload que actúa de puente seguro.

Por qué importa: separar UI de privilegios es el modelo de seguridad de Electron. Romperlo (activar nodeIntegration) expone tu app a ejecución de código si cargas contenido remoto.

Diagrama del flujo

B
Lee el diagrama: el renderer nunca llama a Node; invoca window.api.* que el preload expuso, y eso viaja por un canal IPC hasta el main, que responde.

Main vs Renderer de un vistazo

I

Cuándo: pon en el main todo lo que toque disco, red privilegiada o nativo; deja en el renderer solo la interfaz.

05

Preload & contextBridge

I → A · seguridad

El puente seguro

I

El preload corre antes de tu web, con acceso a ipcRenderer. Expone una API mínima en window con contextBridge — nunca todo Node.

Regla de oro: expón funciones concretas, no ipcRenderer entero ni require. Así el renderer solo puede usar los canales que tú decides.

contextIsolation & aislamiento

A

Valores por defecto seguros desde Electron 12+. No los desactives para cargar contenido remoto.

OpciónQué hace
contextIsolation aísla el preload del JS de la página
nodeIntegration off = el renderer no ve Node
sandbox limita el preload al SO
Por qué: con aislamiento, un XSS en tu UI no obtiene acceso a fs ni a procesos del SO.

08

Empaquetado, auto-update & seguridad

A · producción

Empaquetar con electron-builder

A

Genera instaladores por plataforma desde un bloque build en package.json.

builder vs Forge: electron-builder es config declarativa + auto-update fácil; electron-forge integra empaquetado y plugins en un CLI oficial (npm run make). Elige uno.

Auto-update

A
Cómo: publica releases (p. ej. en GitHub); el updater compara versiones, descarga en segundo plano e instala al reiniciar. Requiere apps firmadas para funcionar sin avisos del SO.

Checklist de seguridad

A
  • contextIsolation: true y nodeIntegration: false (los defaults).
  • Activa sandbox en el renderer.
  • Expón una API mínima por contextBridge; nunca ipcRenderer entero.
  • Valida cada canal y argumento en el main; no confíes en el renderer.
  • Carga solo contenido de confianza; usa HTTPS.
  • Limita la navegación: bloquea will-navigate y window-open a destinos no permitidos.
  • Define una Content-Security-Policy.
  • Mantén Electron al día: parchea Chromium/Node.
  • Abre enlaces externos con shell.openExternal, no en una BrowserWindow.
Regla mental: trata tu propio renderer como si fuera código no confiable. Toda la potencia vive en el main, detrás de canales acotados.
