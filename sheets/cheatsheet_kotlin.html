inferencia

val, var & tipos

B

val = solo lectura (inmutable), tu opción por defecto. var si vas a reasignar. El tipo se infiere; decláralo tras : si quieres ser explícito.

Por quéPreferir val hace el código más predecible y seguro en concurrencia. Usa var solo cuando de verdad muta.

Null-safety — el sello de Kotlin

I

Un tipo no admite null salvo que añadas ?. El compilador te obliga a manejar el caso null, así matas los NullPointerException en tiempo de compilación.

Evita !!!! renuncia a la seguridad y revienta si es null. Resérvalo para cuando estés 100% seguro; casi siempre ?. + ?: o let es mejor.
01

Funciones

ciudadanas
de primera clase

Declarar funciones

B

fun nombre, parámetros con nombre: Tipo y el retorno tras :. Con cuerpo de expresión (=) el tipo se infiere.

Args por defecto / nombrados

B

Valores por defecto eliminan los overloads; los argumentos nombrados hacen la llamada legible y saltan parámetros.

Lambdas & orden superior

I

Una lambda es código entre { }. Una función de orden superior recibe o devuelve funciones. Si la lambda es el último arg, va fuera del paréntesis (trailing).

Funciones de extensión

I

Añade métodos a tipos existentes sin heredar ni tocar su código. Dentro, this es el receptor. Base de muchas APIs y DSL de Kotlin.

02

Clases y objetos

OOP
+ azúcar

Clases y constructores

B

El constructor primario va en la cabecera. val/var ahí declaran propiedades. No existe new.

HerenciaClases y métodos son final por defecto. Marca open para poder heredar o sobrescribir.

data class

I

Para guardar datos. Genera gratis equals, hashCode, toString, copy y la desestructuración.

sealed & enum

A

sealed = jerarquía cerrada y conocida. Con when el compilador comprueba que cubres todos los casos (sin else). Perfecto para estados de UI.

object & companion

I

object = singleton (una sola instancia). companion object da miembros "estáticos" ligados a la clase (factories, constantes).

03

Flujo de control

casi todo
es expresión

when

B

El switch con superpoderes: empareja valores, rangos (in) y tipos (is). Es una expresión: devuelve valor.

if como expresión

B

No hay operador ternario: if/else ya devuelve un valor, así que cumple el mismo papel.

Rangos y bucles

B

Los rangos (.., until, downTo, step) hacen los bucles legibles. Recorre cualquier cosa iterable con for (x in ...).

Scope functions — let · run · with · apply · also

A

Ejecutan un bloque sobre un objeto. Se distinguen por el receptor (this o it) y por qué devuelven (el resultado del bloque o el propio objeto). Elige según para qué.

FnReceptorDevuelveCuándo
letitresultadonull-safety, transformar
runthisresultadoconfig + calcular un valor
withthisresultadoagrupar llamadas a un obj
applythisel objetoconfigurar y devolverlo
alsoitel objetoefectos colaterales / log
Regla rápida¿Devuelves el objeto configurado? apply. ¿Un cálculo derivado? let/run. ¿Solo un efecto (log)? also.
05

Coroutines — concurrencia

la firma
de Kotlin

suspend · launch · async

I

Una corutina es una tarea ligera que se puede suspender sin bloquear el hilo. Una suspend fun solo se llama desde otra corutina o suspend. launch = lanzar (sin resultado); async = lanzar y obtener un valor con await().

Dispatchers

I

Deciden en qué hilo corre la corutina. Elige según el tipo de trabajo.

DispatcherPara
MainUI (Android) — tocar vistas
IOred, disco, base de datos
DefaultCPU intensivo (cálculos)
Unconfinedcasos especiales (raro)
ReglaTrabajo bloqueante → IO. Cálculo pesado → Default. Actualizar UI → Main.

withContext

I

Cambia de hilo dentro de una suspend y vuelve solo al terminar. Lo normal para mover el trabajo pesado fuera de Main.

Structured concurrency

A

Toda corutina vive dentro de un scope. El scope espera a sus hijas y, si se cancela (o una falla), cancela a todas. Resultado: sin fugas ni corutinas huérfanas.

coroutineScope { · · · }
async { uno() }

hija 1 — corre en paralelo

async { dos() }

hija 2 — corre en paralelo

el scope espera a ambas · si una falla → cancela el resto

Cold streams

I

Un Flow emite una secuencia de valores en el tiempo (lo que suspend es a "un valor"). Es cold: el bloque flow { } no se ejecuta hasta que alguien hace collect, y se reinicia con cada colector.

A

Hot flows: emiten aunque nadie escuche. StateFlow guarda un valor actual (estado de UI); SharedFlow retransmite eventos.

ViewModel + coroutines

I

El ViewModel mantiene el estado y sobrevive a rotaciones. viewModelScope lanza corutinas que se cancelan solas al destruirse.

Jetpack Compose

I

UI declarativa: describes qué mostrar con funciones @Composable. Cuando el estado cambia, Compose recompone lo afectado.

Interop con Java

A

Kotlin compila a la JVM y es 100% interoperable: usa cualquier librería Java y, al revés, anota tu Kotlin para que Java lo consuma cómodo.

Anotación / detallePara
@JvmStaticmétodo estático real en Java
@JvmOverloadsgenera overloads de defaults
@JvmFieldexpone campo sin getter
String!tipo "plataforma" de Java
Cuidado con nullLo que viene de Java puede ser null aunque su tipo no lo diga (String!). Trátalo como anulable (?) en la frontera.

collectAsState — del Flow a la UI

I

El puente entre un StateFlow del ViewModel y Compose. collectAsStateWithLifecycle() recoge solo cuando la pantalla está visible (ahorra batería).

El flujo completoViewModel expone un StateFlow · una corutina lo actualiza · Compose lo recoge con collectAsState… y recompone la UI. Esa es la arquitectura reactiva de Android moderno.
