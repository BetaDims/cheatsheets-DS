Lógica: SI, Y, O

SI(prueba; si_verdadero; si_falso). Combina con Y/O para varias condiciones. Para muchos casos encadenados, mejor SI.CONJUNTO (ver Hoja 02) que anidar muchos SI.

Multiplica rangos elemento a elemento y suma el total. Sirve para promedios ponderados, conteos con varias condiciones y cálculos que evitan columnas auxiliares.

BUSCARV y BUSCARH

BUSCARV busca un valor en la primera columna y devuelve algo de la misma fila. El 4º argumento FALSO = coincidencia exacta (úsalo casi siempre). BUSCARH es la versión horizontal.

LímiteBUSCARV sólo mira a la derecha y se rompe si insertas columnas. Para casos nuevos usa BUSCARX o ÍNDICE+COINCIDIR.

BUSCARX — la búsqueda moderna

Reemplaza a BUSCARV: busca en cualquier dirección, no depende de números de columna y trae un valor por defecto si no encuentra. Disponible en Microsoft 365 / 2021+.

=BUSCARX(qué; rango_búsqueda; rango_resultado; si_no_existe)
ESfx=BUSCARX("Mouse";A2:A4;B2:B4;"N/D")copiar
ENfx=XLOOKUP("Mouse",A2:A4,B2:B4,"N/D")copiar

ÍNDICE + COINCIDIR

La combinación clásica antes de BUSCARX, aún imbatible: COINCIDIR halla la posición de un valor; ÍNDICE devuelve lo que hay en esa posición. Busca hacia la izquierda y en dos dimensiones.

Traer el precio buscando por nombre (búsqueda a la izquierda)
ESfx=ÍNDICE(B2:B4;COINCIDIR("Mouse";A2:A4;0))copiar
ENfx=INDEX(B2:B4,MATCH("Mouse",A2:A4,0))copiar
El 0 importaEl 0 de COINCIDIR exige coincidencia exacta; con 1 o -1 busca aproximada (requiere datos ordenados).

SI.CONJUNTO y SI.ERROR

SI.CONJUNTO evalúa varias condiciones en orden y devuelve el primer caso verdadero — más legible que muchos SI anidados. SI.ERROR envuelve cualquier fórmula para mostrar algo limpio si da error.

Clasificar por tramos
ESfx=SI.CONJUNTO(B2>=18;"A";B2>=14;"B";VERDADERO;"C")copiar
ENfx=IFS(B2>=18,"A",B2>=14,"B",TRUE,"C")copiar
Evitar #N/D o #¡DIV/0!
ESfx=SI.ERROR(BUSCARV(A2;Datos;2;FALSO);"No encontrado")copiar
ENfx=IFERROR(VLOOKUP(A2,Datos,2,FALSE),"No encontrado")copiar

Rangos con nombre

Dale un nombre a una celda o rango (Fórmulas → Administrador de nombres o el Cuadro de nombres): =TasaIGV*Base se lee mucho mejor que =$F$1*B2 y no se desordena al copiar.

CuándoPara constantes (tasas, tipos de cambio) y rangos que reutilizas en muchas fórmulas. Los nombres son absolutos por defecto.

Tablas dinámicas

La herramienta más potente para resumir miles de filas sin una sola fórmula: arrastra campos a Filas, Columnas, Valores y Filtros. Insertar → Tabla dinámica.

  1. Parte de una Tabla (Ctrl+T) para que crezca sola al añadir datos.
  2. Filas = categorías · Valores = qué medir (suma, conteo, promedio, % del total).
  3. Segmentadores y escala de tiempo la vuelven interactiva (botones de filtro).
  4. Agrupa fechas por mes/año y números por tramos (clic derecho → Agrupar).
  5. Campo calculado para métricas nuevas (p. ej. Margen = Ganancia/Ventas).
RefrescarLos datos no se actualizan solos: Alt + F5 (esta) o clic derecho → Actualizar. IMPORTARDATOSDINAMICOS / GETPIVOTDATA extrae un valor concreto a una celda.

Visualización & dashboards

intermedio–avanzado
gráficos · KPIs · diseño

Qué gráfico usar (y cuál evitar)

El gráfico correcto responde a una pregunta. Regla general: barras para comparar, líneas para tendencia, dispersión para relación. El circular sólo con 2–3 partes.

Quiero mostrar…Gráfico
Comparar categoríasColumnas / barras (barras si las etiquetas son largas)
Evolución en el tiempoLíneas (o áreas para acumulados)
Composición / partes de un todoBarra apilada 100% > circular
Relación entre dos variablesDispersión (XY)
Dos magnitudes distintasCombo (columnas + línea, eje secundario)
Aporte hacia un totalCascada (waterfall)
EvitaEfectos 3D, demasiados colores y ejes que no empiezan en 0 en gráficos de barras: distorsionan la lectura.

Gráficos dinámicos e interactividad

Un gráfico dinámico se alimenta de una tabla dinámica y se filtra con los mismos segmentadores. Conecta un segmentador a varias tablas para que todo el dashboard reaccione a la vez.

  1. Crea la tabla dinámica → Insertar gráfico dinámico.
  2. Insertar → Segmentación de datos y Escala de tiempo para filtros tipo botón.
  3. Clic derecho en el segmentador → Conexiones de informe para enlazarlo a otras tablas.

Minigráficos (sparklines)

Gráficos diminutos dentro de una celda: tendencia por fila sin ocupar espacio. Insertar → Minigráficos (líneas, columnas o ganancia/pérdida). Perfectos junto a una tabla de KPIs.

Tarjetas KPI

Una cifra grande + comparación vs meta + un símbolo de dirección. Se arma con TEXTO para el formato y formato condicional o SI para el color/flecha.

Variación con flecha y signo
ESfx=SI(C2>=B2;"▲ ";"▼ ")&TEXTO(C2/B2-1;"0.0%")copiar
ENfx=IF(C2>=B2,"▲ ","▼ ")&TEXT(C2/B2-1,"0.0%")copiar

Gráfica de cascada (waterfall)

Muestra cómo se llega de un valor inicial a uno final sumando y restando pasos (ventas → costos → gastos → utilidad). En 365/2016+: Insertar → Cascada; marca los subtotales como Total.

Diseño de dashboard — principios

  1. Separa capas: una hoja de datos crudos, una de cálculo, una de presentación. No mezcles.
  2. Una idea por vista. Lo más importante arriba a la izquierda (donde cae la vista).
  3. Menos es más: pocos colores, alineación a una rejilla, números formateados.
  4. Interactividad con segmentadores; protege las hojas de fórmulas para que nadie las rompa.
  5. Un clic para actualizar: centraliza con tablas dinámicas / Power Query y un botón de Actualizar todo.

Hoja 04

Power tools

avanzado
Power Query · Power Pivot · DAX

Power Query — limpiar y combinar (ETL)

El "antes de la fórmula": importa de archivos/carpetas/web/BD, limpia y transforma con pasos grabados que se repiten con un clic al actualizar. Datos → Obtener datos. Cada paso queda registrado en el lenguaje M.

  1. Obtener datos (archivo, carpeta entera, web, base de datos).
  2. Transforma: quitar columnas, cambiar tipos, dividir, reemplazar, rellenar, quitar duplicados.
  3. Anexar (apilar tablas iguales) o Combinar/Merge (cruzar por una clave, como BUSCARV).
  4. Cerrar y cargar → a una Tabla o al Modelo de datos. Al llegar datos nuevos: Actualizar.
NotaEl lenguaje M es el mismo en cualquier idioma de Office (no se traduce) y distingue mayúsculas.

Power Pivot — modelo de datos

Permite cargar millones de filas y relacionar varias tablas (ventas ↔ calendario ↔ productos) sin BUSCARV. Activa el complemento y construye relaciones en la vista de diagrama.

  1. Carga las tablas al Modelo de datos (desde Power Query o "Agregar al modelo").
  2. Crea relaciones 1-a-muchos por la columna clave (clic y arrastra en el diagrama).
  3. Usa una tabla calendario dedicada para la inteligencia de tiempo.
  4. Escribe medidas en DAX y úsalas en tablas dinámicas sobre el modelo.

DAX esencial

El lenguaje de las medidas de Power Pivot/Power BI. Una columna calculada se evalúa fila a fila y ocupa memoria; una medida se calcula al vuelo según el contexto del filtro (casi siempre quieres medidas). DAX no se traduce: mismas funciones en inglés.

CALCULATEEs la función central: cambia el contexto de filtro de una medida. DIVIDE evita el error de dividir entre cero.

M vs DAX — cuándo cada uno

Ejemploslimpiar, dividir, combinar, anexarKPIs, % del total, YTD, vs año anterior

Automatización · Macros & VBA

avanzado
grabar · VBA · automatizar

Grabar una macro y abrir el editor

Una macro repite acciones automáticamente. Grábala (Vista → Macros → Grabar) para que Excel escriba el VBA por ti, luego límpialo. El editor se abre con Alt + F11. Guarda como .xlsm (libro habilitado para macros).

EmpezarGraba la tarea una vez, mira el código generado y edítalo: es la forma más rápida de aprender VBA.

Estructura: Sub, variables y tipos

El código vive en módulos. Un Sub ejecuta acciones; una Function devuelve un valor. Option Explicit al inicio te obliga a declarar variables (evita erratas). Declara con Dim … As Tipo.

TipoGuarda
Stringtexto
Integer · Longenteros (usa Long por seguridad)
Doubledecimales
BooleanTrue / False
Range · Worksheetobjetos de Excel

Rangos y objetos

VBA maneja Excel por objetos: Workbooks → Worksheets → Range/Cells. Cells(fila, columna) es ideal dentro de bucles. With evita repetir el objeto.

Decisiones: If y Select Case

Bucles

For…Next cuando sabes cuántas veces; For Each para recorrer una colección; Do While/Until mientras se cumpla una condición (datos de largo desconocido).

Interacción: MsgBox e InputBox

MsgBox muestra un mensaje; InputBox pide un dato al usuario. Úsalos para confirmar acciones o capturar parámetros.

Funciones personalizadas (UDF)

Una Function en un módulo se usa como cualquier fórmula en la hoja. Ideal para reglas propias (IGV, comisiones).

Botones y manejo de errores

Asigna una macro a un botón (Insertar → Formas o control de formulario → clic derecho → Asignar macro). Protege el código con On Error.

VelocidadPara macros largas: Application.ScreenUpdating = False al inicio y True al final acelera mucho la ejecución.

Finanzas: VNA, TIR y el valor del dinero

Evalúan proyectos descontando flujos futuros. Cuidado: VNA asume que el primer flujo ocurre al final del periodo 1 — por eso la inversión inicial (periodo 0) se suma aparte.

TIR sobre todos los flujos (incluida la inversión en B1)
ESfx=TIR(B1:B6)copiar
ENfx=IRR(B1:B6)copiar

Modelado: supuestos, escenarios y equilibrio

Un buen modelo separa supuestos (celdas de entrada, en color) de cálculos. Cambias los drivers y todo se recalcula. La sensibilidad se hace con Tabla de datos; el escenario base/optimista/pesimista con el Administrador de escenarios.

Punto de equilibrio (uds) = Costos fijos ÷ (Precio − Costo variable unitario)
Buscar objetivo¿Qué precio necesito para una utilidad de 50 000? Datos → Análisis de hipótesis → Buscar objetivo, fijando la celda de utilidad.

Forecasting (pronóstico de series de tiempo)

Elige el método según el patrón: media móvil suaviza ruido; suavizado exponencial da más peso a lo reciente (factor α); regresión capta tendencia; modelos con estacionalidad para patrones que se repiten.

Suavizado exponencial: Ft+1 = α·At + (1 − α)·Ft   (0 < α < 1)
Pronóstico lineal del próximo punto
ESfx=PRONOSTICO.LINEAL(x;ConocidosY;ConocidosX)copiar
ENfx=FORECAST.LINEAR(x,ConocidosY,ConocidosX)copiar

Inventarios: EOQ, POQ y Pareto

Modelos clásicos de gestión de stock. El EOQ halla el tamaño de pedido que minimiza el costo total (pedir + mantener). Pareto/ABC separa los pocos artículos vitales (≈20% que mueven ≈80%).

EOQ:   Q* = √( 2 · D · S / H )

D = demanda anual · S = costo de pedir · H = costo de mantener por unidad/año.

Estructura del libro

  1. Separa capas: datos crudos → cálculos → presentación, en hojas distintas. Una sola fuente de la verdad.
  2. Entradas en color: marca las celdas de supuestos para distinguirlas de las que llevan fórmula. Nunca escribas sobre una fórmula.
  3. De arriba-izquierda hacia abajo-derecha: los cálculos fluyen en una dirección, sin referencias circulares.
  4. Una hoja "Léeme": versión, fecha, autor y supuestos clave.

Tablas y nombres en vez de rangos fijos

Usa Tablas (Ctrl+T) y referencias estructuradas: =SUMA(Ventas[Importe]) se expande sola y se lee mejor que =SUMA(B2:B999). Pon las constantes en celdas con nombre, no dentro de la fórmula.

Por quéLas referencias estructuradas y los nombres sobreviven a inserciones de filas/columnas; los rangos fijos, no.

Robustez de fórmulas

  1. No "hardcodees": nada de números mágicos dentro de fórmulas; ponlos en una celda de parámetros.
  2. Envuelve riesgos con SI.ERROR y valida con ESNUMERO/ESERROR (ISNUMBER/ISERROR).
  3. Referencias absolutas correctas (F4) al arrastrar; revisa con un par de celdas antes de copiar a miles.
  4. Evita columnas/filas enteras (A:A) en fórmulas pesadas: ralentizan el libro.

Separador de argumentos: ; vs ,

En esta hoja las fórmulas en español usan punto y coma ; y las de inglés usan coma ,. Cuál te toca depende de la configuración regional de tu equipo (en muchas instalaciones en español el separador es ;, en inglés/EE. UU. es ,).

Si una fórmula da error al pegarlaCambia ; por , (o al revés) según tu Excel. También cambia el separador decimal (coma vs punto). Lo ajustas en Archivo → Opciones → Avanzadas → Separadores del sistema.
