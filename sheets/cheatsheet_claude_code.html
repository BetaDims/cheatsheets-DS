¿Qué es Claude Code?

B

Claude Code es un agente de programación que vive en tu terminal: lee tu repo, edita archivos, ejecuta comandos y corre tests por ti, dentro de un bucle plan → actuar → verificar. No es autocompletado: es un colaborador que entiende todo el proyecto.

Agénticousa herramientas (leer, editar, bash) en bucle hasta terminar.
Con permisospide tu OK antes de acciones que cambian cosas.
Dóndeterminal, VS Code, JetBrains, app de escritorio y web.

Instalar & arrancar

B

Instala el CLI y lanza claude dentro de la carpeta de tu proyecto. La primera vez te pide iniciar sesión.

Modos de trabajo — Shift+Tab rota

B

Claude Code tiene varios modos que cambian cuánta autonomía le das. Cicla entre ellos con Shift+Tab.

ModoComportamiento
Normalpide permiso para cada edición / comando
Auto-acceptaplica ediciones sin preguntar (sigue acotado)
Plansolo investiga y propone un plan; no toca nada
RecomendaciónEmpieza en Plan para tareas grandes: revisas el plan, lo apruebas y entonces ejecuta. Menos sorpresas.

El ciclo agéntico

B

Le das un objetivo en lenguaje natural; Claude explora el código, planifica, edita y ejecuta, y verifica (tests, lint). Repite hasta cumplir o pedirte algo.

prompt → explorar → editar → verificar

Planificar antes de tocar

I

Para cambios grandes, pídele un plan primero. Revisas, ajustas y solo entonces ejecuta. Evita refactors a ciegas.

Buenos prompts

I
Específicodi el qué, el porqué y la restricción, no solo "arréglalo".
Da contextoapunta a los archivos con @ en vez de describirlos.
Verificablepide que corra los tests y enseñe la salida.
Iteracorrige sobre la marcha; no rehagas el prompt entero.

CLAUDE.md — memoria del proyecto

I

Un archivo que Claude lee siempre al arrancar: convenciones, comandos, estilo. Créalo con /init y edítalo a mano.

NivelesHay CLAUDE.md de proyecto (compartido en git) y ~/.claude/CLAUDE.md personal (tus preferencias en todos los repos).

Gestionar la ventana de contexto

I

El contexto es finito. Limpia o compacta cuando cambies de tarea para que Claude no arrastre ruido viejo.

ComandoQué hace
/clearborra la conversación; empieza limpio
/compactresume el historial para liberar contexto
/contextmuestra cuánto contexto se está usando
/costtokens y coste de la sesión
ReglaUna tarea = una conversación. /clear entre tareas distintas evita confusiones y ahorra tokens.

MCP — conectar herramientas

I

El Model Context Protocol conecta Claude a herramientas externas (bases de datos, GitHub, Figma…) vía servidores. Una vez añadidos, Claude puede usarlos como tools.

Hooks — disparar acciones

A

Los hooks ejecutan comandos tuyos en eventos del ciclo (antes/después de una tool, al terminar…). Útil para formatear o validar automáticamente. Van en settings.json.

Subagentes & skills

A

Un subagente es un asistente con su propio rol, prompt y herramientas, que Claude invoca para tareas acotadas (revisar, explorar). Vive en .claude/agents/.

Para quéaislar contexto y paralelizar (revisión, búsqueda).
Gestionarcon /agents.

Una skill es una carpeta con un SKILL.md que aporta conocimiento especializado bajo demanda: su descripción está siempre, el detalle se carga cuando hace falta.

Para quéflujos y best-practices reutilizables.
Pluginsempaquetan comandos, agentes y skills juntos.

B

Claude redacta mensajes de commit y abre PRs por ti (usa git y gh por debajo). Pídeselo en prosa.

I

Permisos

A

Controla qué puede hacer Claude sin preguntar: una allowlist en settings.json reduce interrupciones sin abrir la mano del todo.

YOLO con cabeza--dangerously-skip-permissions salta todas las preguntas: úsalo solo en entornos aislados (un contenedor), nunca sobre tu máquina con secretos.

Headless & SDK

A

Con -p lo ejecutas sin interfaz, ideal para scripts y CI. El Agent SDK deja construir tus propios agentes sobre el mismo motor.

Elegir modelo

I

Cambia el modelo con /model según la tarea. Más capacidad para lo difícil; más velocidad/coste para lo simple.

ModeloCuándo
Opus 4.8lo más capaz: refactors grandes, trabajo agéntico largo
Sonnet 4.6equilibrio velocidad/inteligencia para el día a día
Haiku 4.5rápido y económico para tareas simples
Fast mode/fast activa salida más rápida con Opus (no baja a un modelo menor). Disponible en Opus 4.8/4.7/4.6.
